# API 清理执行总结（历史归档）

本文件为历史归档：用于记录早期“无版本前缀 API”清理工作的思路与结论。

为避免误导集成方，本文件不再列出任何旧的无版本前缀 API 端点（这些端点在当前版本中不应被新集成使用，且部分已移除）。

## 当前以代码为准的权威来源

- OpenAPI：`/openapi.yaml` 或 `/openapi.json`
- Swagger UI：`/docs`
- 路由注册：以后端实际路由注册为准

## 当前集成规则（摘要）

- 业务 API 使用版本前缀：`/api/v1/...`
- 健康检查：`/api/health`
- 非托管原则：后端不接收助记词/私钥；客户端本地管理密钥与签名；后端只接收地址/公钥等公开信息，以及已签名交易/交易记录等。

## 你可能在找的内容

- 当前 API 详单与示例：请看同目录的 `API_REFERENCE.md`，以及 `/openapi.yaml`。
